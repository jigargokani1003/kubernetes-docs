Topic 1Setup

KIND Cluster Setup

KIND (Kubernetes IN Docker) runs real K8s nodes as Docker containers. 3 tools + 1 config = full multi-node cluster in 5 minutes. Zero cloud cost.

📦 Install All 3 Tools
bash — Linux/WSL
# 1. Docker
sudo apt-get update && sudo apt-get install -y docker.io
sudo systemctl enable --now docker
sudo usermod -aG docker $USER   # log out + back in after this!

# 2. kubectl
curl -LO "https://dl.k8s.io/release/$(curl -sL https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
chmod +x kubectl && sudo mv kubectl /usr/local/bin/
kubectl version --client

# 3. KIND
curl -Lo ./kind https://kind.sigs.k8s.io/dl/v0.22.0/kind-linux-amd64
chmod +x kind && sudo mv kind /usr/local/bin/
kind --version
🏗️ Multi-Node Cluster Config
kind-config.yaml
kind: Cluster
apiVersion: kind.x-k8s.io/v1alpha4
nodes:
  - role: control-plane
    extraPortMappings:        # expose ports for Ingress
    - containerPort: 80
      hostPort: 80
    - containerPort: 443
      hostPort: 443
  - role: worker
  - role: worker
bash — Create & verify
kind create cluster --name my-cluster --config kind-config.yaml
kubectl get nodes          # 3 nodes = Ready
kubectl cluster-info
kind get clusters
kind get nodes --name my-cluster
CommandPurposeExample
kind create clusterCreate clusterkind create cluster --name dev --config kind.yaml
kind get clustersList clusterskind get clusters
kind delete clusterDestroy clusterkind delete cluster --name dev
kind load docker-imagePush local image inkind load docker-image myapp:v1 --name dev
kind export kubeconfigGet kubeconfigkind export kubeconfig --name dev
🧠 Memory Trick

D-K-K = Docker → Kubectl → KIND. Like building a house: foundation (Docker) → tools (kubectl) → blueprint+structure (KIND cluster). Can't skip steps!

⚠️
Common Mistake: Forgetting sudo usermod -aG docker $USER then log out + log in. Without this, all KIND commands fail with "permission denied".

Pods — The Atomic Unit
A Pod is the smallest deployable unit. Wraps one or more containers that share the same IP, network namespace, and volumes. Every container in K8s lives inside a Pod.

ReplicaSets — Self-Healing
A ReplicaSet ensures N identical pods are always running. Kill one — it immediately creates a replacement. The supervisor that never sleeps.

  

    
⚡ Self-Healing Simulator

    

    

    

      
      
      
      
      
    

  

  

  
CommandPurposeExample

  
kubectl get rsList ReplicaSetskubectl get rs -o wide

  
kubectl describe rsEvents + owned podskubectl describe rs nginx-rs

  
kubectl scale rsChange replicaskubectl scale rs nginx-rs --replicas=5

  
kubectl delete rsRemove RS + podskubectl delete rs nginx-rs

  

  
🧠 Memory Trick
ReplicaSet = Security Guard doing headcount every few seconds. Too few? Spawn more. Too many? Remove extras. Always hits the exact target number.

  
💡
Best Practice: Don't use ReplicaSets directly. Use Deployments — they wrap ReplicaSets and add rolling updates + rollback history.

Services
Pods get new IPs every restart. A Service gives a stable virtual IP + DNS name that load-balances across matching pods. Without Services, nothing can reliably reach your pods.

  

    
🌐 Service Types — Interactive

    

      
      
      
    

    

      
    

  

  
CommandPurposeExample

  
kubectl get svcList serviceskubectl get svc -o wide

  
kubectl exposeCreate service from resourcekubectl expose deploy nginx --port=80 --type=NodePort

  
kubectl describe svcEndpoints + selectorskubectl describe svc nginx

  
kubectl get endpointsPod IPs behind servicekubectl get endpoints nginx

  
kubectl create svc --dry-runGenerate YAMLkubectl create svc clusterip nginx --tcp=80:8080 --dry-run=client -o yaml

🧠 Memory Trick
C-N-L = Club, Neighborhood, Lobby. ClusterIP = Club (pods only). NodePort = Neighborhood (same network). LoadBalancer = Hotel Lobby (public). Pick based on who needs access.

Topic 7Config
Namespaces
Divide one cluster into virtual clusters. Same hardware, different teams — dev / staging / prod isolated by default.

  

  
bash


kubectl get namespaces
kubectl create namespace dev
kubectl apply -f pod.yaml -n dev
kubectl get pods -n dev
kubectl get pods -A
kubectl config set-context --current --namespace=dev
# Cross-namespace: svc.namespace.svc.cluster.local

  
🧠 Memory Trick
Namespace = Office Floor. Same building (cluster), different floors (namespaces). Talk across floors using full address: service.namespace.svc.cluster.local

  
⚠️
kubectl delete namespace production deletes EVERYTHING inside — pods, services, secrets, configs. No confirmation. Be very careful!

Topic 8Config
ConfigMaps
Decouple non-sensitive config from container images. Same Docker image, different behavior in dev vs prod — just swap the ConfigMap.

  

  
bash


kubectl create configmap app-config \
  --from-literal=LOG_LEVEL=debug \
  --from-literal=DB_URL=postgres://db:5432/app \
  --dry-run=client -o yaml > cm.yaml
kubectl apply -f cm.yaml
kubectl get configmap
kubectl describe configmap app-config
kubectl create configmap file-config --from-file=app.conf

  
🧠 Memory Trick
ConfigMap = .env file for K8s. Don't hardcode config in Docker images — externalize it so you can change behavior without rebuilding the image.

Topic 9Security
Secrets
Store sensitive data (passwords, tokens, TLS certs) separately from code. Base64-encoded in etcd with RBAC protection. Use encryption-at-rest in production.

  

  
bash


kubectl create secret generic db-secret \
  --from-literal=username=admin \
  --from-literal=password=Sup3rS3cr3t!
kubectl get secret db-secret -o yaml
kubectl get secret db-secret -o jsonpath='{.data.password}' | base64 -d
kubectl create secret tls my-tls --cert=tls.crt --key=tls.key

  
🧠 Memory Trick
ConfigMap = Plain envelope. Secret = Sealed envelope. Base64 is NOT encryption — it just hides from casual glance. In production always enable etcd encryption-at-rest!

Topic 10Storage
Volumes & Persistent Volumes
Container filesystem is ephemeral — restart = data gone. PV is cluster-level durable storage. PVC is your pod's claim to that storage.

  

  
pv-pvc.yaml


apiVersion: v1
kind: PersistentVolume
spec:
  capacity:
    storage: 1Gi
  accessModes: [ReadWriteOnce]
  hostPath:
    path: /data/pv1

  
🧠 Memory Trick
PV = Parking Lot. PVC = Parking Ticket. Pod = Car. Claim a spot (PVC), the lot gives a real space (PV). Your car (pod) doesn't care which exact spot.

Topic 11Networking
Ingress
One external IP, many backends. Route api.myapp.com → backend, app.myapp.com → frontend. Replaces N LoadBalancer services with one controller.

  

  
ingress.yaml


# Install NGINX Ingress for KIND
kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/main/deploy/static/provider/kind/deploy.yaml
---
apiVersion: networking.k8s.io/v1
kind: Ingress
spec:
  ingressClassName: nginx
  rules:
  - host: myapp.local
    http:
      paths:
      - path: /api
        pathType: Prefix
        backend:
          service:
            name: api-service
            port:
              number: 80

  
🧠 Memory Trick
Ingress = Hotel Concierge. One front desk (controller), routes guests to rooms (services). You tell the concierge the path (/api, /app) — they handle the rest.

Topic 12Scale
Autoscaling — HPA
HPA watches CPU/memory. Spikes → add pods. Drops → remove pods. Sleep through Black Friday. Set it and forget it.

  

    
📈 HPA Simulator — Drag Load

    

      CPU Load:
      
      30%
      Threshold: 70%
    

    

    

  

  
hpa.yaml


# Quick imperative
kubectl autoscale deployment nginx --cpu-percent=70 --min=2 --max=10
kubectl get hpa --watch

# Declarative
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: nginx
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70

  
🧠 Memory Trick
HPA = Elastic waistband. Sets min (won't go skinnier) and max (won't burst). Expands when you eat more (load), contracts when you don't. Always comfortable.

  
⚠️
HPA requires resources.requests.cpu on containers. Without it, HPA can't calculate utilization and stays inactive forever.

Topic 14Operations
Debugging Kubernetes
K8s always tells you what's wrong — you just need to ask the right questions in the right order. Follow the decision tree below to solve 99% of issues.

  

  
ErrorRoot CauseFix

  
CrashLoopBackOffApp crashes on startkubectl logs --previous

  
ImagePullBackOffBad image / no authFix image name, add imagePullSecret

  
OOMKilledMemory limit exceededIncrease resources.limits.memory

  
Pending (no resources)No node has capacitykubectl describe pod → Events

  
Endpoints: noneService selector mismatchAlign service.selector with pod.labels

  
ErrImageNeverPullImage not in clusterkind load docker-image myapp:v1

  
EvictedNode out of disk/memorykubectl describe node → Conditions

  

  
bash — G-D-L-E-E Debug Loop


# G - Get: what's broken?
kubectl get pods -A | grep -Ev 'Running|Completed'
# D - Describe: why is it broken?
kubectl describe pod <name>   # Events section is gold!
# L - Logs: what did the app say?
kubectl logs <name> --previous 2>&1 | tail -50
# E - Exec: can I get inside?
kubectl exec -it <name> -- /bin/sh
# E - Events: what happened cluster-wide?
kubectl get events --sort-by=.lastTimestamp -A | tail -20
# Bonus: debug networking
kubectl run net --image=nicolaka/netshoot -it --rm --restart=Never

  
🧠 Memory Trick
G-D-L-E-E = Get → Describe → Logs → Exec → Events. Five commands. The answer to every K8s problem is in one of these five. Tattoo it on your wrist.
